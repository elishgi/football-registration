# ⚽ הרשמה לכדורגל

MVP בעברית, RTL ו-Mobile First לניהול הרשמה למשחק כדורגל קבוע. שחקנים נרשמים ללא התחברות, מנהל מנהל משחקים/קבועים/תשלומים, והמערכת מחשבת אוטומטית מי בפנים ומי ממתין.

## יכולות

- דף ציבורי `/game/[eventId]` ודף בית שמפנה למשחק הפעיל הראשון.
- דף מנהל `/admin` עם סיסמה יחידה דרך `ADMIN_PASSWORD`.
- Supabase כמסד נתונים חינמי ופשוט.
- ניהול משחק: שם, תאריך, שעה, מקסימום שחקנים, דדליין תשלום, פתיחה/סגירה.
- ניהול שחקנים קבועים.
- הרשמת קבועים מרשימת בחירה והרשמת לא-קבועים לפי שם.
- מניעת כפילות שם באותו משחק כל עוד ההרשמה לא בוטלה.
- ביטול הגעה עם אישור בדף הציבורי.
- סימון תשלום ידני על ידי מנהל בלבד.
- העתקת רשימה מוכנה לוואטסאפ.

## חוקי קדימות

לפני דדליין התשלום: קבועים לפי זמן הרשמה, ואז לא-קבועים לפי זמן הרשמה.

אחרי דדליין התשלום: קבועים ששילמו, לא-קבועים ששילמו, קבועים שלא שילמו בזמן, לא-קבועים שלא שילמו בזמן. בתוך כל קבוצה הסדר הוא לפי זמן ההרשמה המקורי.

המערכת תמיד לוקחת את `max_players` הראשונים כ-`ACTIVE`; כל השאר `WAITING`. לכן אין מצב של פחות מ-15 בפנים ועדיין יש ממתינים, אלא אם המקסימום הוגדר אחרת.

## הקמה מקומית

```bash
npm install
cp .env.example .env.local
```

מלאו את הערכים ב-`.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
ADMIN_PASSWORD=change-me
```

צרו פרויקט Supabase חדש, פתחו את SQL Editor והריצו את `supabase/schema.sql`.

לאחר מכן:

```bash
npm run dev
```

פתחו `http://localhost:3000/admin`, התחברו עם `ADMIN_PASSWORD`, צרו משחק והוסיפו שחקנים קבועים.
